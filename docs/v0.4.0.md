# Ziron v0.4.0 Roadmap

**Target Release**: Enhanced directory recognitions and context awareness

This document outlines the features planned for Ziron v0.4.0, focusing on expanding directory recognition capabilities to automatically detect and display various development contexts, version control systems, container environments, and cloud provider configurations.

---

## Enhanced Git Module

### Git Branch & Status Improvements
- [x] Enhanced branch display with remote tracking information
- [x] Show ahead/behind status relative to remote branch
- [x] Display current commit hash (short format)
- [x] Show tag information if on a tag
- [ ] Display merge/rebase status
- [x] Show stash count
- [ ] Performance optimization for large repositories

### Git Status Details
- [x] Show number of modified files
- [x] Show number of staged files
- [x] Show number of untracked files
- [x] Show number of conflicts (during merge/rebase)
- [ ] Display rebase/merge progress
- [ ] Show submodule status

### Git Remote Information
- [x] Display remote name (origin, upstream, etc.)
- [ ] Show remote URL (abbreviated)
- [ ] Display fetch/push status

## Performance Targets

- Directory context detection: < 10ms (with caching)
- Module data fetching: < 50ms per module (with cache)
- Prompt rendering with multiple modules: < 5ms
- Memory usage: < 40MB (idle, with all modules loaded)
- CPU usage: < 1% (idle)

---

## Breaking Changes

None planned for v0.4.0. This release focuses on adding new modules and enhancing existing ones while maintaining backward compatibility with v0.3.0.

---

## Migration Notes

Users upgrading from v0.3.0 to v0.4.0:
- No configuration changes required
- Existing themes remain compatible
- New modules are opt-in (must be added to configuration)
- Git module enhancements are backward compatible
